# 98 签到验证码微调按钮设计

## 目标

在人工验证码页面中，为 `slide` 和 `rotate` 两种验证码增加精确微调能力，同时保持现有验证码获取、答案格式、提交、校验、重试和签到流程不变。

## 交互设计

- `slide`：图块拖动区域下方显示“左移 1px / 右移 1px”按钮。
  - 每次点击只调整验证码原图坐标系中的 `left` 1 像素。
  - 继续通过现有 `render()` 生成 `x,y` 答案，其中 `y` 保持 `dy`。
  - 在边界 `0` 与 `masterW - tw` 处钳制，不越界。
- `rotate`：角度滑条旁显示“左转 1° / 右转 1°”按钮。
  - 每次点击调整滑条值 1 度，范围保持 `0..359`。
  - 继续通过现有 `renderAngle(true)` 生成角度答案并同步旋转预览。
- 按钮使用 `type="button"`，避免触发表单或改变现有提交行为。
- `click` 及其他验证码类型完全不变。

## 安全边界

- 不修改验证码接口、会话状态、TTL、浏览器上下文、FlareSolverr、Cloudflare 处理、答案校验和签到请求。
- 不引入新的答案格式或额外转换。
- 微调按钮只调用现有答案渲染路径，保证视觉状态、页面显示和最终 `answer` 同步。

## 验证

- 静态回归测试确认按钮只出现在 `slide`/`rotate` 模板分支。
- 确认 slide 微调调用 clamp + render，最终仍为 `x,y`。
- 确认 rotate 微调同步 range 并调用 renderAngle，最终仍为角度字符串。
- 运行 Python 语法编译和 JSON/版本一致性检查。
