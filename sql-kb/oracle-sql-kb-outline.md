# Oracle SQL 知识库第一版提纲

## 定位

这是面向用户工作场景的第一版 Oracle SQL 知识库提纲，不是最终文档，也不是完整数据字典。目标是先把这批历史脚本沉淀为可持续更新的知识底座，后续支持自然语言到 SQL / 脚本生成。

## 语料边界

- 当前输入以工作中实际使用的 Oracle SQL 为主。
- 辅助 Python 文件暂不纳入核心知识范围，默认视为结果加工或文件处理脚本。
- 该语料库不是一次性导入，后续按增量更新处理。

## 第一版结构

### 1. 业务域地图

按脚本内容先建立业务域，而不是先建表字典。当前优先识别和沉淀：

- 流失 / 携出 / 解约追踪
- 升套 / 外呼 / 合约 / 维系任务
- B2I / 客群运营 / 渗透率分析
- 宽带 / 融合 / FTTR / 套餐营销
- 名单型取数 / 策略筛选 / 中间表生成

### 2. 核心数据资产层

建立第一版高频表与职责映射：

- 用户主表 / 用户派生标签表
- 产品 / 套餐 / 行业卡 / 物联网剔除相关表
- 出账 / 费用 / 收入相关表
- 携转 / 携出 / 解约相关表
- 状态 / 用户标签 / 合约标签相关表
- 宽带 / 融合 / 活动相关表

目标不是一次性做完整字段字典，而是先识别：

- 高频核心表
- 每张表的大致用途
- 哪些表常一起 join
- 哪些表在不同业务域里复用

### 3. 规则知识层

这一层优先沉淀注释里明确出现的业务口径，而不仅是 SQL 语法：

- 剔除规则
- 保留规则
- 时间口径
- 入网时长口径
- 合约到期 / 次月追踪口径
- 套餐推荐 / 档位归集规则
- 风险客群 / 托底逻辑 / 例外条件

这部分后续会成为自然语言写 SQL 时最关键的背景。

### 4. SQL 模式层

沉淀高频查询模式，而不是只按文件归档：

- 名单筛选型 SQL
- 汇总指标型 SQL
- 对比分析型 SQL（上月 / 当月 / 次月）
- 追踪型 SQL（解约后状态、携出后状态）
- 中间表 / 临时表构建型 SQL
- 推荐策略标签生成型 SQL

后续每类模式都可以抽出可复用模板。

### 5. Oracle 风格层

明确这套语料的 SQL 风格偏好，后续生成时对齐：

- 大量使用 `to_char`, `to_date`, `add_months`
- 常见 `nvl`, `decode`, `case when`
- 使用分析函数，如 `row_number() over (...)`, `partition by`
- 强依赖按月/按日分区字段（如 `month_id`, `day_id`）
- 以中间表、临时表、create table as select 为重要工作流

后续生成 SQL 时默认优先复用这种风格，而不是切换成其他数据库习惯。

### 6. 输出物层

后续知识库分阶段沉淀为：

- 业务域总览
- 高频表清单
- 高频规则手册
- 常用 SQL 模板
- 自然语言需求到 SQL 的提示规则
- 版本更新日志

## 当前阶段已经确认的事实

- SQL 文件数量远高于 Python 文件，语料主体明确。
- 这批脚本主要服务真实业务任务，不是零散练习文件。
- 文件内注释包含大量关键业务规则，知识价值很高。
- 存在明显的高频表、稳定写法、固定任务主题。
- 适合做成持续演进的工作知识库。

## 下一阶段计划

### 阶段 2：业务域细化

把现有脚本按业务主题重新归档，输出更稳定的分类树，而不是沿用原目录名。

### 阶段 3：高频表认知

为高频表补充：

- 表用途
- 典型 join 对象
- 常见过滤字段
- 常见输出字段
- 出现在哪些业务域

### 阶段 4：规则抽取

从注释和 SQL 条件里抽出规则，整理成“口径层知识”。

### 阶段 5：模板化

整理成可复用模板，支持后续自然语言生成：

- 名单筛选模板
- 指标分析模板
- 解约追踪模板
- 升套任务模板
- 携出分析模板

### 阶段 6：Skill 化

在知识结构稳定后，将其封装成一个面向工作场景的 Oracle SQL assistant skill。

## 增量更新原则

- 新脚本进入后，不从头重学。
- 先判断属于哪个业务域。
- 再判断是否引入新表、新规则、新模板。
- 对已有规则做补充或修正，而不是重复记录。
- 保留明显的版本变更信息，尤其是时间口径和剔除规则变化。

## 最终目标

让后续工作方式变成：

- 用户用自然语言描述需求
- 助手根据已有业务知识、规则和 Oracle 风格生成 SQL
- 必要时再补充脚本、导出逻辑或中间表构建步骤

而不是每次从零猜测业务口径。
